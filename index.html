★ Catatan Belajar

Berikut hal-hal penting yang sudah dipelajari di Web Client Development:

  1. Struktur HTML → setiap halaman web wajib punya DOCTYPE, html, head, dan body.
  2. Tag formatting teks seperti bold, italic, underline, highlight, maupun strong.
  3. Perbedaan <b> dan <strong> → keduanya tebal, tapi strong punya makna semantik.
  4. Cara membuat hyperlink ke google dengan tag <a href> dan menyisipkan gambar dengan <img src>.
  5. Penggunaan inline style untuk font, color, dan spacing.

Topik yang masih perlu diperdalam:

  • CSS Selector → terutama perbedaan class dan ID.
  • Box Model → memahami margin, padding, dan border secara visual.
  • Konsep specificity → aturan mana yang menang saat ada konflik style.

✎ Refleksi Pengerjaan

Approach

Dalam pengerjaan ini tentunya pada awalnya saya membuat file index terlebih dahulu dan menghubungkann folder yang berisi file itu ke github sebagai repository salah satu syarat tugas ini. Baru setelah itu saya mulai mengerjakan tugas ini dengan membuat struktur HTML terlebih dahulu seperti yang sudah dipelajari di kelas pada pertemuan - pertemuan sebelumnya, Dengan mengikuti contoh yang ada di drive saya membuat struktur HTML yang terdiri dari tag DOCTYPE, html, head, dan body. Setelah itu saya mulai mengisi bagian body dengan konten yang sesuai dengan tugas ini seperti identitas mahasiswa, daftar mata kuliah, catatan belajar, dan refleksi pengerjaan. Saya juga menggunakan tag-tag HTML seperti table untuk membuat tabel jadwal mata kuliah, ol untuk membuat daftar berurutan catatan belajar, dan ul untuk membuat daftar tidak berurutan topik yang perlu diperdalam. Selain itu saya juga menggunakan inline style untuk memberikan beberapa gaya pada teks seperti warna dan spasi. Adapaun untuk beberapa simbol seperti Bintang dan Garis horizontal saya mencari Numeric Character Reference yang sesuai dengan contoh yang ada untuk menyisipkan simbol tersebut ke dalam halaman web, adapun beberapa simbol seperti pensil itu saya mencari di internet untuk bentukan pensil yang sesuai dengan contoh. Setelah selesai membuat konten dan struktur HTML, saya melakukan pengecekan kembali untuk memastikan tidak ada kesalahan penulisan tag atau struktur yang salah. Terakhir saya menyimpan file index.html dan menguploadnya ke repository github yang sudah saya buat sebelumnya.
